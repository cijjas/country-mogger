# Agent notes

Guidance for AI coding agents working in this repository.

## Project map

- `app/` Next.js App Router shell (layout, page, global styles, favicon)
- `components/map-explorer.tsx` React orchestrator: state, URL sync, header card, hints
- `components/map/renderer.ts` imperative D3 renderer (SVG, pin, tooltip, zoom, fills); talks to React only through the `MapRenderer` / `RendererCallbacks` interfaces in `lib/types.ts`
- `components/comparison-panel.tsx` the analysis card (number, flag donut, contributor list)
- `lib/geo/fill-engine.ts` pure fill algorithm (flood fill, water jumps, partials)
- `lib/geo/polygon.ts` pure screen-space polygon math (organic cut, projections)
- `lib/geo/load-world.ts` topology fetch + precomputation into `GeoData`
- `lib/metrics.ts` metric definitions, formatting, groups, flag helpers
- `lib/metric-data.ts` GENERATED by `scripts/build-data.mjs`; never edit by hand
- `tests/` Vitest unit tests for the pure modules

## Commands

`npm run dev` | `build` | `lint` | `typecheck` | `test` | `build-data`

CI (`.github/workflows/ci.yml`) runs lint, typecheck, test and build; all four must pass.

## Conventions

- Keep `lib/geo/` pure (no DOM, no React) so it stays unit-testable.
- New metrics must be additive totals; convert rates via population in the build script.
- UI copy: plain sentences, no em dashes.
- The React/D3 boundary is intentional: React never touches the SVG, the renderer never calls React APIs directly.
